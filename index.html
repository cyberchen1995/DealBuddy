01三件套

一套三件，各管一段。

不是一个大而全的 App，而是三个各司其职的小件，装完都归你。

Capture

Chrome 采集扩展

你打开商品页，点一下「整理当前商品信息」，标题、页面价、SKU、店铺、规格和详情图文字就成了一条带采集时间的记录。

边界只处理你正打开的页面，OCR 在浏览器本地跑。

Workbench

本地 Web 工作台

一个只监听 127.0.0.1 的小服务：会话、商品事实表、报告、追问区都在这里，数据落在本机 ~/.dealbuddy。

边界不连外部服务；LLM 是可选项，配了才启用。

Agent Skill

Agent Skill + MCP

本地 MCP endpoint 加上 skills/dealbuddy 集成材料，让 Codex 等支持 MCP 的 agent 直接调用你的选品会话继续研究。

边界MCP 只接受 localhost 来源的请求。

02怎么用

照常逛店，顺手存证，回来看报告。

全程不改变你逛店的方式——DealBuddy 只负责把你看过的页面变成可比较的事实。

  1. 第一幕

    照常逛店

    在淘宝、天猫、京东打开候选商品页。找货这件事还是你来——DealBuddy 不代逛、不代点，也不替你做决定。

  2. 第二幕

    顺手存证

    点一下扩展里的「整理当前商品信息」：可见字段几秒钟进工作台，详情长图里的规格文字由本地 OCR 随后补全——图片不出你的浏览器。

  3. 第三幕

    出报告，再追问

    工作台把候选商品并排成事实表，生成 Markdown 选品报告；想深挖就在追问区继续。默认用本地规则回答，接不接 LLM 由你决定。

03报告样张

每个结论，
都能倒查。

  • 价格分层写。划线价、页面展示价、页面可见优惠、估算应付各归各位；「估算应付」只计入页面能直接解析的优惠，不冒充结算价格。

  • 记录带出处。每条商品都保留来源链接、SKU、采集与复核时间、数据可信度，报告是可追溯的事实摘要，不是拍脑袋的结论。

  • 纯 Markdown。复制进笔记、聊天窗口或另一个 agent 都不失真，证据跟着结论一起走。

04底线

丑话说在前面。

这些「不做」写进了代码，也写在这里。装之前先看清楚，省得装完失望。

隐私，四个不做

  • 不接平台 API。只读你打开的页面上可见的内容。
  • 不导出 Cookie。OCR 下载商品图时也不携带网站 Cookie。
  • 不绕验证码。遇到风控就停下，不做任何规避。
  • 不碰下单结算。购物车、订单、支付页面一概不进。

另外两件小事：商品图的 OCR 在你的浏览器本地执行，图片不上传；LLM 默认关闭，只有你配置了 provider，追问才会对外发请求，界面会标明当前 provider。

价格，三句实话

  • 报告只写「估算应付」。不写「到手价」，最终价格以平台订单结算页为准。
  • 不承诺最低价。只承诺每个数字有出处、有采集时间。
  • 不抢券、不批量爬。只整理你亲自打开的页面，不做自动化批量采集。

完整的数据处理说明都在隐私政策里，一共十条，五分钟能读完。

05常见疑问

装之前，
大概会想问的。

没找到答案就去 GitHub Issues 提一个。

我的数据到底存在哪？

会话与报告默认保存在本机 ~/.dealbuddy/sessions，配置在 ~/.dealbuddy/config.json。工作台只监听 127.0.0.1，没有云端、没有账号体系。

必须配置 LLM 才能用吗？

不用。排序、报告、追问默认由本地规则完成。只有你在设置里填了 provider 和 key，追问区才会调用外部模型，并且界面会显示当前 provider。API Key 只写入本机配置文件。

它会不会在后台自动抓数据？

不会。扩展只处理你当前打开的商品详情页：默认需要你点「整理当前商品信息」；也可以为逛店过程开启自动整理，但它同样只作用于你正在看的页面，随时可关，不存在批量爬取。

支持哪些平台？

当前支持淘宝、天猫、京东的商品详情页。工作台里可以把三个平台的商品放进同一个会话并排比较。

安装麻烦吗？要什么环境？

扩展从 GitHub Releases 下载 zip，用 Chrome 开发者模式加载；工作台需要 Python 3.12 和 uv，两条命令启动。完整步骤见上手三步。目前通过 GitHub Releases 分发。

06上手

三步，装完就是你的。

开源 MIT，没有订阅，没有账号。需要本机 Chrome、Python 3.12 与 uv。

  1. 1

    下载并解压采集扩展

    从 GitHub Releases 获取 dealbuddy-capture-0.3.0.zip（约 19 MB），解压到任意目录。

    下载 dealbuddy-capture-0.3.0.zip
  2. 2

    用开发者模式加载扩展

    打开 chrome://extensions → 右上角开启「开发者模式」→「加载已解压的扩展程序」→ 选择刚才解压的目录。

  3. 3

    启动本地工作台

    克隆仓库，两条命令把工作台跑起来：

    $ git clone https://github.com/cyberchen1995/DealBuddy.git && cd DealBuddy
$ uv sync && uv run dealbuddy web --port 8765

    浏览器打开 http://127.0.0.1:8765，输入品类和需求，开逛。
