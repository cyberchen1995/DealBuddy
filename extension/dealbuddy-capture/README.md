# DealBuddy 采集扩展

Chrome MV3 扩展，DealBuddy 购物搭子的唯一商品数据来源。在淘宝/天猫/京东商品详情页
提取标题、价格、店铺、SKU、规格与详情图，并用本地 PP-OCR 补全长图里的文字，随后
自动投递到本机 DealBuddy 工作台。

## 安装

1. 打开 `chrome://extensions`，启用「开发者模式」。
2. 点「加载已解压的扩展程序」，选择本目录 `extension/dealbuddy-capture/`。
3. 启动工作台：`uv run dealbuddy web --port 8765`，并在工作台创建/选择会话。

## 使用

- **手动**：在商品详情页点扩展图标 → 「整理当前商品信息」。
- **自动**：popup 里开启「自动采集」，进入受支持的商品详情页即自动执行。
- 采集完成后数据自动 POST 到 `http://127.0.0.1:8765/api/current/offers`
  （可在 popup 修改），工作台会在数秒内自动同步。

采集过程会短暂滚动页面以触发详情懒加载，**结束后自动还原你的滚动位置**。

## 设置（popup）

| 设置 | 说明 |
|------|------|
| 自动采集 | 进入商品详情页自动整理（同一商品同一 SKU 只执行一次） |
| 详情图等待超时 | 懒加载等待上限，默认 12s |
| 投递地址 | 本机 DealBuddy 接收端点，默认 `/api/current/offers` |

## 隐私边界

- OCR 完全在浏览器本地（隐藏 iframe + onnx/wasm）执行，图片不出本机。
- 不读取 Cookie、不导出登录态、不绕验证码、不碰购物车/结算。
- 数据只发往你配置的 `127.0.0.1` 本机端点。

## 排错

| 现象 | 处理 |
|------|------|
| 「发送到本机 DealBuddy 失败」 | 确认 `dealbuddy web` 在跑、端口一致；可先在 popup「复制 JSON」兜底 |
| 详情图很少/规格为空 | 天猫规格多在长图里，确认 OCR 完成（popup 状态）；重采一次命中缓存会更快 |
| OCR 慢 | 首次需加载模型（已与等图并行预热）；识别结果按图片 URL 缓存，重复采集近乎即时 |
| 页面不支持 | 仅支持淘宝/天猫/京东**商品详情页**，搜索页/店铺页不触发 |
